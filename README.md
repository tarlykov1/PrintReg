# GSPLabelPrinter

GSPLabelPrinter — локальное Windows-приложение для поиска сотрудников в CSV и печати именных наклеек 40 × 60 мм. Приложение запускает локальный ASP.NET Core Minimal API только на `http://127.0.0.1:5187`, открывает браузер и не использует базу данных.

## Статус автоматической сборки

Windows-сборка проверяется GitHub Actions workflow `.github/workflows/windows-build.yml` на `windows-latest`: restore, build Release x64, test Release x64, self-contained publish win-x64, проверка структуры публикации и smoke test опубликованного EXE.

> Важно: локальная сборка считается подтверждённой только после успешного прохождения Windows workflow или локального запуска команд на машине с .NET SDK 8 и Windows.

## Скачивание Windows-сборки из GitHub Actions

1. Откройте вкладку **Actions** в GitHub.
2. Выберите последний успешный workflow **Windows build**.
3. Скачайте artifact **GSPLabelPrinter-win-x64**.
4. Распакуйте архив в отдельную папку, например `C:\GSPLabelPrinter`.
5. Запустите `GSPLabelPrinter.exe`.

## Первый запуск

При первом запуске приложение создаёт рядом с EXE:

- `config.json`;
- `data/employees.csv`;
- `backup/`;
- `logs/`.

Если `employees.csv` отсутствует, создаётся демонстрационный файл. Рабочие пути рассчитываются от фактической папки EXE, поэтому запуск двойным щелчком, из PowerShell, через ярлык и из каталогов с пробелами/кириллицей должен использовать одну и ту же папку приложения.

## Запуск в режиме разработки

```powershell
scripts/run-development.ps1
```

Или вручную:

```powershell
dotnet restore GSPLabelPrinter.sln
dotnet run --project src/GSPLabelPrinter/GSPLabelPrinter.csproj -p:Platform=x64
```

## Публикация

```powershell
scripts/publish-win-x64.ps1
```

Скрипт выполняет restore, build, test, publish и проверяет итоговую структуру. Результат:

```text
output/GSPLabelPrinter-win-x64
```

В папке публикации должны быть `GSPLabelPrinter.exe`, `config.json`, `data/employees.csv`, `backup/`, `logs/`, `wwwroot/` и frontend-файлы.

## CSV

Файл по умолчанию: `data/employees.csv`.

```csv
ФИО;Должность
Иванов Иван Иванович;Главный специалист
Петрова Анна Сергеевна;Начальник отдела
```

Поддерживаются UTF-8 с BOM и без BOM, кириллица, пустые строки, кавычки, переносы строк и символ `;` внутри значения. При записи сохраняется UTF-8 with BOM.

## Настройка принтера

1. Откройте страницу **Настройки**.
2. Выберите установленный Windows-принтер.
3. Проверьте ориентацию, поля, размеры шрифтов и количество копий.
4. Нажмите **Сохранить**.
5. Выполните **Тестовую печать**.

Если принтер не выбран, кнопка печати на главной странице блокируется. Приложение не отправляет задания на случайный принтер.

## Проверка физического размера наклейки

После тестовой печати измерьте наклейку линейкой. Ожидаемый размер — 40 × 60 мм. В коде размер переводится в сотые доли дюйма: 40 мм ≈ 157, 60 мм ≈ 236.

## Что делать, если печатается A4

1. Откройте свойства принтера в Windows.
2. Создайте пользовательскую форму бумаги 40 × 60 мм.
3. Выберите эту форму в драйвере принтера.
4. Отключите масштабирование, «Подогнать под страницу» и похожие опции.
5. Повторите тестовую печать.

Если драйвер не принимает пользовательский размер, приложение вернёт сообщение: «Драйвер принтера не принял размер наклейки 40 × 60 мм...».

## Что делать, если текст смещён

- Проверьте ориентацию в настройках приложения.
- Уменьшите поля в миллиметрах.
- Проверьте физические отступы в драйвере принтера.
- Убедитесь, что драйвер не масштабирует и не центрирует задание самостоятельно.

## Что делать, если интерфейс не открывается

- Проверьте, что приложение запущено.
- Откройте вручную `http://127.0.0.1:5187`.
- Проверьте папку `logs` рядом с EXE.
- Убедитесь, что папка `wwwroot` присутствует рядом с EXE в опубликованной сборке.

## Что делать, если порт 5187 занят

Приложение запишет ошибку в журнал и покажет Windows-сообщение. Освободите порт 5187 или завершите другой экземпляр GSPLabelPrinter. Повторный запуск уже работающего приложения не стартует второй сервер, а открывает существующую страницу.

## Что делать, если CSV открыт в Excel

Закройте Excel и повторите сохранение. При блокировке файла приложение показывает понятное сообщение и не удаляет существующий CSV.

## Восстановление из резервной копии

Перед каждым реальным изменением CSV создаётся backup в `backup/` с именем вида:

```text
employees_2026-06-30_073015_1234567.csv
```

Для восстановления:

1. Закройте приложение.
2. Скопируйте нужный файл из `backup/`.
3. Замените им `data/employees.csv`.
4. Запустите приложение.

## Обновление приложения без потери employees.csv

**Перед обновлением приложения сохраните папки `data` и `backup`, а также файл `config.json`.**

Скрипт публикации не перезаписывает существующий рабочий `data/employees.csv` в папке `output/GSPLabelPrinter-win-x64`, если файл уже существует. При ручном обновлении также не удаляйте `data/`, `backup/` и `config.json`.

## Конфигурация

Пример находится в `config.example.json`. Если рабочий `config.json` повреждён, приложение создаёт `config.json.broken_yyyy-MM-dd_HHmmss`, пишет ошибку в журнал и восстанавливает настройки по умолчанию.

## Логи

Логи находятся в `logs/app-yyyy-MM-dd.log`. Файлы старше 30 дней удаляются автоматически.
